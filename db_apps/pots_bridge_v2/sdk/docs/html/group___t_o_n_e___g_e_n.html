Detailed Description

This group of functions is related to playing out general tones - such as dial tone, busy tone, etc. One would typically call ProSLIC_ToneGenSetup first followed by ProSLIC_ToneGenStart and after some time, a call to ProSLIC_ToneGenStop to stop the tone. The direction of the tone and cadence (if any) are configured using the GUI configuration tool's TONE dialog box and then saved in the constants file.

Function Documentation

int ProSLIC_ToneGenSetup ( proslicChanType_ptr  hProslic,
int  preset 
)

Configure the tone generator to the given preset (see your constants header file for exact value to use). It does NOT play a particular tone out.

Warning
If ProSLIC_FSKSetup was called earlier, this function will need to be called again.
Parameters
[in]hProslic- which channel to configure
[in]preset- which of the xxxx_Tone_Cfg structures to use to configure the channel.
Return values
int- error from errorCodeType RC_NONE indicates no error.
Note
This function can be compiled out by defining DISABLE_TONE_SETUP.
Typically, if one is using the ProSLIC for tone generation, you will be calling this function prior to each call into ProSLIC_ToneGenStart so that the correct tone is played out.

int ProSLIC_ToneGenStart ( proslicChanType_ptr  hProslic,
uInt8  timerEn 
)

This function starts the tone configured by ProSLIC_ToneGenSetup. It is assumed that the Audio setup was performed prior to calling this function. Also, it is suggested that ProSLIC_ToneGenSetup be called if FSK/Caller ID generation functions were used.

Parameters
[in]hProslic- which channel should play the tone.
[in]timerEn- are the timers to be enabled? 1 = Yes, 0 = No. When in doubt, set to 1.
Return values
int- error from errorCodeType RC_NONE indicates no error.
