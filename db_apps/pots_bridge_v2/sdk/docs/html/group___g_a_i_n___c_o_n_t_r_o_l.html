int ProSLIC_AudioGainSetup ( proslicChanType_ptr  hProslic,
int32  rxgain,
int32  txgain,
int  preset 
)

Configures and sets the audio gains - for both RX (toward the phone) and the TX (toward the network). Unlike the ProSLIC_RXAudioGainSetup and ProSLIC_TXAudioGainSetup this does both the calculations and then configures the given channel in one step. It is assumed that the preset gain array is at least 2 elements in size.

Parameters
[in]hProslic- which channel to configure
[in]rxgain- what is the gain/loss of the RX transmit path in 1 dB units (negative = attenuation)
[in]txgain- what is the gain/loss of the TX transmit path in 1 dB units (negative = attenuation)
[in]preset- what is the impedance preset value
Return values
int- error from errorCodeType RC_NONE indicates no error.

The following code * fragment will set the RX audio gain to -1 dB snd TX audio gain to -5 dB:

1 ProSLIC_AudioGain(myChannel, -1, -5, MY_IMPEDENCE);

int ProSLIC_RXAudioGainScale ( proslicChanType_ptr  hProslic,
int  preset,
uInt32  pga_scale,
uInt32  eq_scale 
)

Adjusts gain of RX audio path by scaling the PGA and Equalizer coefficients by the supplied values.

Parameters
[in]hProslic- which channel to configure
[in]preset- which preset to use (this may be from the constants file)
[in]pga_scale- scale factor which pga coefficients are to be multiplied
[in]eq_scale- scale factor which equalizer coefficients are to be multiplied
Return values
int- error from errorCodeType RC_NONE indicates no error.
Note
This function can be used in lieu of ProSLIC_dbgSetRXGain to dynamically change the audio gain when gain steps of 0.1dB do not suffice, or if an attenuation

30dB is required.

The following code * fragment will scale the TX audio gain by a factor of (0.707)*(0.500)

1 uInt32 pga_gain_scale = 707;
2 uInt32 eq_gain_scale = 500;
3 ProSLIC_RXAudioGainScale(pProslic,DEFAULT_PRESET,pga_gain_scale,eq_gain_scale);

int ProSLIC_RXAudioGainSetup ( proslicChanType_ptr  hProslic,
int  preset 
)

Configures the RX audio gain (toward the telephone). This function DOES NOT actually calculate the values for the preset. To dynamically calculate the values, you would need to call ProSLIC_dbgSetRXGain .

Parameters
[in]hProslic- which channel to configure
[in]preset- which preset to use (this may be from the constants file)
Return values
int- error from errorCodeType RC_NONE indicates no error.
Note
One can use this in conjunction with ProSLIC_dbgSetRXGain to dynamically change the audio gain. It is important to have the presets of both routines match.

The following code * fragment will set the RX audio gain to -1 dB:

1 ProSLIC_dbgSetRXGain(myChannel, -1, MY_IMPEDENCE,0);
2 ProSLIC_RXAudioGainSetup(myChannel,0);
